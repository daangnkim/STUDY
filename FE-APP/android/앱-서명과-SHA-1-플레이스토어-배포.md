# 앱 서명 · SHA-1 · 플레이스토어 배포 이해하기

> opical 앱(Capacitor 웹뷰 앱) 플레이스토어 배포를 준비하며 정리한 학습 노트.

## TL;DR (한 문장)

> **도장(서명키)으로 ① 앱에 서명하고, 그 도장의 지문(SHA-1)을 ② 여러 서비스에 등록한다. 그런데 "Play 앱 서명" 때문에 도장이 "내 업로드 키 + 구글의 앱 서명 키" 2개라서, 지문도 2개고 상황에 맞는 걸 골라 써야 한다.**

---

## 1. 등장인물 3명 (도장 / 금고 비번 / 지문)

앱 서명을 **인감도장**에 비유하면 딱 맞는다.

| 실제 파일/값 | 비유 | 정체 | 공개해도 되나? |
|---|---|---|---|
| `xxx.jks` (키스토어) | **인감도장(실물)** | 진짜 비밀. 이걸로 앱에 도장을 찍음 | ❌ 절대 비공개 |
| 키스토어 비밀번호 | **도장을 넣어둔 금고 비번** | 도장을 꺼내 쓰려면 필요한 암호 | ❌ 절대 비공개 |
| `7B:3C:33:...` (SHA-1) | **그 도장이 찍는 무늬(지문)** | "내 도장은 이렇게 생겼다"는 식별표 | ✅ 공개해도 안전 |

- **SHA-1은 도장 자체가 아니라, 그 도장이 찍은 자국의 고유한 무늬다.**
- 사람마다 지문이 다르듯, 도장마다 SHA-1이 다르다.
- 남에게 보여줘도 그걸로 도장을 복제할 수 없어서 안전하다.

---

## 2. 도장으로 하는 일은 2가지

### (A) 앱에 직접 도장 찍기 = "서명(signing)"
- 앱 설치파일(AAB/APK)에 도장을 찍는 행위.
- "이 앱은 내가 만든 진짜다. 위조 아니다"를 증명.
- 👉 필요한 것: **도장 실물(.jks) + 금고 비번**. (SHA-1 아님)
- 👉 빌드할 때 `build.gradle`의 signingConfig 설정으로 일어남.

### (B) 다른 서비스에 "내 도장 무늬는 이거야" 신고 = "지문(SHA-1) 등록"
- 카카오, 네이버, 구글 같은 외부 서비스에 미리 알려주는 것.
- "앞으로 이 무늬(SHA-1)로 찍힌 앱만 진짜 내 앱이니, 그 앱에서 오는 요청만 받아줘."
- 👉 필요한 것: **지문(SHA-1)** 뿐. 도장 실물은 안 줌.

**핵심:** 도장 실물은 내 빌드 컴퓨터에만 있고, 지문(SHA-1)만 여기저기 등록한다.

---

## 3. SHA-1(지문)을 요구하는 곳이 여러 곳인 이유

여러 곳에서 요구하지만 이유는 전부 **"위조 앱 차단"** 으로 동일하다.

| 요구하는 곳 | 왜 SHA-1을 달라고 하나 |
|---|---|
| **카카오 로그인** | "이 무늬 앱에서 온 로그인만 허용" (가짜 앱이 카카오 로그인 못 쓰게) |
| **네이버 로그인** | 위와 동일 |
| **구글 로그인 / 지도 / Firebase** | 위와 동일 |
| **Play Console** | 구글이 "이 앱의 공식 도장이 맞는지" 관리하려고 |

---

## 4. ⚠️ 핵심 — 도장이 사실 "2개"다 (Play 앱 서명)

요즘 플레이스토어는 **"Play 앱 서명(Play App Signing)"** 이라는 안전장치를 쓴다. 그래서 도장이 둘로 나뉜다.

```
 [나]                              [구글]                        [사용자 폰]
  │                                  │                              │
  │  ① 업로드 키로 도장 찍어 제출  →   │  ② 구글이 진짜 도장          │
  │     (= 업로드 키 .jks)            │     (앱 서명 키)으로          │
  │                                  │     다시 찍어서 배포     →     │  설치되는 진짜 앱
```

- **업로드 키(upload key)** = 내가 가진 도장(내가 만든 `.jks`). 구글 현관에 들어갈 때 쓰는 **출입증**.
- **앱 서명 키(app signing key)** = 구글이 보관하는 진짜 배포용 도장. 실제 사용자 폰에 깔리는 앱에 최종으로 찍힘.

**왜 이렇게 번거롭게?**
- 내 업로드 키를 잃어버려도 구글에서 재발급 받을 수 있어 안전하다.
- (옛날엔 도장 하나라 잃어버리면 앱을 영영 업데이트 못 했다.)

**그 대가로 SHA-1이 2개가 된다:**

| 도장 | SHA-1 | 어디서 얻나 | 어디에 쓰나 |
|---|---|---|---|
| **업로드 키** | 내가 만든 키의 지문 | 내 컴퓨터에서 `keytool`로 추출 | 개발/테스트 단계 |
| **앱 서명 키** | 구글이 만든 키의 지문 | **Play Console**(첫 업로드 후 생성) | **실제 배포된 앱** |

---

## 5. ⚠️ 카카오/네이버 로그인 단골 함정

> **"내 폰 테스트는 되는데, 스토어에서 받은 앱은 로그인이 안 됨"**

이유: 카카오에 **업로드 키** 키 해시만 등록했기 때문. 실제 배포 앱은 **앱 서명 키**로 찍히므로 무늬가 달라 카카오가 거부한다.

**해결: 카카오에 키 해시를 2개 등록한다.**
1. **업로드 키** 키 해시 → 개발/테스트 단계에서 동작
2. **앱 서명 키** 키 해시 → 실제 스토어 배포 앱에서 동작 (첫 AAB 업로드 후 Play Console에서 SHA-1 확인 → 키 해시로 변환해 등록)

> 카카오 "키 해시"는 SHA-1을 Base64로 변환한 값이다. (예: `keytool -exportcert ... | openssl sha1 -binary | openssl base64`)

## 6. 플레이스토어 배포 전체 로드맵

```
[1] Google Play Console 개발자 계정 준비      ← 가게 입점 등록 (1회 $25)
[2] 서명 키(keystore) 만들기                  ← 앱의 '인감도장'. 위변조 방지용
[3] 릴리스용 AAB 파일 빌드                     ← 스토어에 올릴 '제품 상자'(.aab)
[4] Play Console에서 앱 등록 + 스토어 정보 입력  ← 이름/설명/스크린샷/개인정보 등
[5] AAB 업로드 → 내부 테스트 → 정식 출시        ← 실제 배포
```

## 8. 자주 쓰는 명령어

```bash
# 키스토어(도장) 만들기
keytool -genkeypair -v -keystore opical-upload-key.jks \
  -alias opical -keyalg RSA -keysize 2048 -validity 10000 -storetype PKCS12

# SHA-1 / SHA-256 지문 보기
keytool -list -v -keystore opical-upload-key.jks -alias opical

# 카카오 키 해시(Base64) 뽑기
keytool -exportcert -alias opical -keystore opical-upload-key.jks \
  | openssl sha1 -binary | openssl base64

# 릴리스 AAB 빌드 (JDK 21)
cd android
JAVA_HOME=$(/usr/libexec/java_home -v 21) ./gradlew bundleRelease
# 결과물: android/app/build/outputs/bundle/release/app-release.aab
```

## 9. 용어 사전

| 용어 | 뜻 |
|---|---|
| **키스토어(keystore, .jks)** | 서명 키(도장)를 담은 파일 |
| **서명(signing)** | 앱 설치파일에 도장을 찍어 진위를 증명하는 것 |
| **SHA-1 / SHA-256** | 인증서(도장)의 지문. 공개해도 안전한 식별값 |
| **키 해시(key hash)** | SHA-1을 Base64로 변환한 값. 카카오 등에서 사용 |
| **업로드 키(upload key)** | 내가 만들어 구글에 제출용으로 쓰는 도장 |
| **앱 서명 키(app signing key)** | 구글이 보관하며 실제 배포 앱에 찍는 진짜 도장 |
| **Play 앱 서명** | 구글이 앱 서명 키를 대신 보관·관리해주는 방식 |
| **AAB(Android App Bundle)** | 플레이스토어에 올리는 앱 패키지 형식(.aab). APK 대체 |
| **alias(별칭)** | 키스토어 안의 특정 키를 가리키는 이름표 |
